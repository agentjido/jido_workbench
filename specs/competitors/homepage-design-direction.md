# Homepage Design Direction (Mastra + LangChain Bias)

Reviewed: 2026-02-25
Scope: Non-GitHub competitor homepages only
Excluded: GitHub-only homepages (`10-pi-mono`, `11-sagents`)

## What to emulate from Mastra

1. Hero that ships value in under 5 seconds
- Direct build verb in headline.
- Immediate quickstart command in hero.
- Tight capability chips (agents/workflows/RAG/tools) right away.

2. Concrete, visual example cards
- Example gallery with recognizable use cases (browser agent, coding, research, text-to-sql).
- Each card explains output and utility, not just technology.

3. Builder-first narrative
- "Build and iterate" then "Deploy and scale" progression.
- Product shown as developer workflow, not abstract promise.

## What to emulate from LangGraph (LangChain)

1. Control + agency positioning
- Strong framing around reliability and orchestration, not generic "AI platform" language.

2. Feature narrative by operating concern
- Human-in-the-loop, workflow control, memory, streaming.
- Each section maps to real production pain points.

3. Trust and proof cadence
- Logos/social proof near top.
- FAQ and final conversion CTA near bottom.

## Recommended homepage element order for Jido

1. Header with product + docs + examples + github + CTA
2. Hero: value prop + quickstart command + primary CTA
3. Trust bar (adoption proof: users, repos, production teams)
4. "What you build" card grid (6 concrete agent examples)
5. "How Jido works" architecture strip (agents, tools, workflows, state, supervision)
6. Reliability section (HITL, observability, testing/evals, recovery)
7. Developer experience section (SDK, templates, local dev, deployment path)
8. Integrations section (LLM providers, data, runtime surfaces)
9. Community/resources section (docs, examples, tutorials)
10. Final CTA (start building + talk to team)
